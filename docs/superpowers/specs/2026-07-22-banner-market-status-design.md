# 배너 시장별 데이터 기준일·상태 표시 — 설계

날짜: 2026-07-22

## 1. 배경

현재 배너는 `meta.built_at`(배치 실행 시각) 하나만 표시한다. 이 때문에:

- **시장별 데이터 날짜를 알 수 없다.** 실측: `rs_asof.US = 2026-07-22`인데 실제
  US 데이터는 `detail/KGS.json → last_date = 2026-07-21`이다.
- **장중/완결을 구분할 수 없다.** 2026-07-22 11:48 KST(한국 장중)에 배치를 돌렸을 때
  미완결 봉이 적재됐는데, 배너는 이를 알릴 수단이 없었다.
- **데이터가 멈춘 것을 알 수 없다.** GitHub Actions는 저장소 60일 무활동 시 예약
  워크플로를 자동 중지한다. 이때 사이트는 그대로 뜨지만 데이터만 낡는다.

## 2. 목표 / 비목표

**목표**
1. 시장별 데이터 기준일 + 상태(최신/장중/지연)를 배너 칩으로 표시
2. 데이터가 오래 갱신되지 않으면 경고
3. 장중일 때 "직전 완결일 기준"임을 안내

**비목표 (YAGNI)**
- 공휴일 캘린더 도입 (아래 3.2에서 공휴일 무관 방식을 채택해 불필요)
- 실시간 시세 표시

## 3. 설계

### 3.1 백엔드 — `rs_asof`를 실제 마지막 봉 날짜로

현재 `run.py`가 `rs_asof = str(trading.latest_complete_date(...))`를 넣는다. 이건
**기대 완결일**이지 실제 데이터 날짜가 아니다. 검증 결과 토요일 19:00 KST 기준
`latest_complete_date('KR')`가 `2026-07-25`(토요일, 비거래일)를 반환한다. 즉 봉이
없는 날짜가 기록될 수 있다.

**변경**: `build_market`이 `frames`의 실제 `max(date)`를 계산해 `rs_asof`로 사용한다.
`smoke.py`의 동일 패턴도 함께 수정한다.

**파급**: `rs_asof`는 현재 생성만 되고 **소비처가 없다**(grep 확인: build.py /
run.py / smoke.py). 따라서 의미 변경으로 깨지는 곳이 없다.

### 3.2 프론트 — 상태 판정 (순수 함수)

`site/banner-status.js`에 순수 함수로 분리한다(브라우저 전역 + Node 모듈 동시 지원).

```
입력: { market: 'US'|'KR', asof: 'YYYY-MM-DD', builtAt: ISO문자열, now: Date }
출력: { state: 'stale'|'intraday'|'fresh', staleDays: number }
       staleDays = built_at 이후 경과한 일수(캘린더 일, 내림). stale이 아니면 그대로 경과일.
```

**판정 규칙 (우선순위 순)**

1. **stale(지연)** — `now - builtAt` 경과가 **3 캘린더일** 초과
   (`market`과 무관한 사이트 전역 판정이므로 모든 시장 칩이 함께 지연으로 표시된다)
2. **intraday(장중)** — 해당 시장이 현재 개장 중
   - KR: 09:00–15:30 `Asia/Seoul`
   - US: 09:30–16:00 `America/New_York` (서머타임은 `Intl` timeZone이 자동 처리)
3. **fresh(최신)** — 그 외

**지연 판정을 `built_at` 경과로 삼는 이유**: `built_at`은 매 실행마다 타임스탬프가
갱신되어 JSON이 항상 달라지고, 따라서 배치가 돌 때마다 커밋된다. 공휴일에도 cron
(월–금)이 실행되므로 `built_at`이 전진한다. → **공휴일 오탐이 없다.** 거래일 갭을
계산하는 방식은 설·추석 연휴에 3~5일 갭이 생겨 거짓 "지연"이 발생하므로 채택하지 않는다.

임계 3일 = 주말 2일 + 여유 1일.

**백엔드 경고와 역할 분담** (중복·모순 방지)
- 프론트 `built_at` 경과 → **사이트 전체가 멈춤** 감지
- 백엔드 `materials_fresh` → **시장별 갱신 실패** (기존 `meta.warnings` 유지)

### 3.3 날짜 포맷 버그 수정

기존 `fmtDate`는 date-only 문자열을 `new Date()`로 UTC 파싱한 뒤 **로컬 게터**로
출력한다. 실측 결과:

| 뷰어 타임존 | `fmtDate("2026-07-21")` |
|---|---|
| Asia/Seoul | 2026-07-21 ✅ |
| America/New_York | **2026-07-20** ❌ |
| America/Los_Angeles | **2026-07-20** ❌ |

칩에는 `new Date`를 쓰지 않고 문자열을 분해하는 전용 `fmtDateOnly`를 사용한다.
(기존 `fmtDate`는 `built_at`(전체 ISO) 표시용으로 유지)

### 3.4 렌더링

- `by_country[market] > 0`인 시장만 칩 표시 (데이터 없는 시장은 백엔드 "건너뜀"
  경고가 대신 안내)
- 상태별 색: 최신 초록 / 장중 노랑 / 지연 빨강
- 장중·지연이면 칩 아래 한 줄 안내 문구
- 기존 `meta.warnings` 렌더링은 그대로 병행 유지
- 반응형: 모바일 폭에서 마스트헤드 줄바꿈 확인
- `index.html` 스크립트 순서: `banner-status.js`를 `app.js` **앞**에 로드

### 3.5 가이드 탭 동기화

배너를 바꾸면 가이드 탭 설명이 사실과 어긋난다. `site/index.html`의 **133, 135행**
(상단 배너 설명)을 새 칩 기준으로 수정한다.

- 226행("종가 — 데이터 기준일의 마지막 가격")은 그대로 유효
- 주석 캡처 이미지는 카드 내부만 담아 배너를 포함하지 않으므로 **재촬영 불필요**

## 4. 테스트

- `banner-status.js` 순수 함수 Node 단위 테스트: 장중 / 마감 후 / 주말 / 지연 경계 /
  서머타임 경계
- 날짜 포맷: 여러 타임존에서 오프바이원 없음 확인
- 기존 `pytest` 회귀 (현재 32개)
- 브라우저 육안 확인(날짜 주입)

**CI 배치 주의**: 테스트를 `daily.yml`에 넣지 않는다. 테스트 실패가 데이터 갱신·배포를
차단해 사이트가 묵게 되기 때문이다. 테스트는 **push 트리거의 별도 워크플로**로 분리한다.

## 5. 리스크 / 전환기

- `rs_asof`를 고쳐도 라이브 US는 **다음 US 배치(07:00 KST)까지 구 라벨(07-22)** 을
  유지한다. 즉시 정확히 하려면 US 배치를 한 번 수동 실행한다.
- 프론트 변경이라 되돌리기는 git revert로 즉시 가능 — 되돌리기 어려운 단계 없음.
